What is MetaQ?

MetaQ is a web-based search platform built for MetaPathways data. It integrates a backend search engine server with indexed annoted metagenomic data and a frontend web visualizing module for both open reading frame annotations as well as microbiological pathway annotations.

How does this demo work?

For this demo, 10 functionally annotated bacterial metagenome samples were indexed, courtesy of the Hallam Lab. This data includes over 2 million individual open reading frames to explore, each annotated against MetaCyc, KEGG and COG protein databases.

MetaQ is still a work in progress, and it currently supports 2 search modules. The GeneExplorer module works for searches based on open reading frames, so one can search functionally annotated genes through protein product names (as default), as well as add filters like a minimum rpkm allowed or for results to require a functionally annotation id (from COG or KEGG) to refine the searches. In addition, the GeneExplorer module also includes a clustering visualization which groups up results based on common keywords between them for explorative purposes. On the other hand, the PathwayExplorer module works around protein pathways. A user can search what identified pathways were annotated from the data (based on their pathway name), and extract the ORFs associated with it, as well as what metagenome sample it was found on.

Current functionality also supports exporting the results found and downloading them as a TSV file with each column attribute for further use.

What is MetaQ built on?

The backend is an Apache Solr server instance, it indexes both ORFs and pathways and serves all the data to the frontend application. This frontend application was built using the Play Framework, which parses user input and communicates with the backend directly. Complimentary data visualization in the frontend is done through JavaScript's D3 library.
